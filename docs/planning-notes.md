# 企画会議メモ

作成日: 2026-07-05

## 現在のコンセプト理解

このゲームの中心は、アイテムを集めるために冒険へ出ること。

現状の実装は、拠点から出撃し、ランダム生成ダンジョンで敵や素材を探し、バッグ容量の範囲で持ち帰る抽出型ローグライクの試作になっている。

## マップ設計の課題

ローグライクとして考えると完全ランダムマップは自然だが、タルコフライク寄りにする場合は問題がある。

欲しいアイテムを集めるゲームなのに、マップも入手場所も完全ランダムだと、プレイヤーが「何を狙ってどこへ行くか」を判断しにくい。

タルコフライク的な面白さには、次の要素が必要。

- 欲しい素材を狙いに行けること
- 危険地帯や安全な帰路を覚えられること
- 脱出口までのルートを計画できること
- マップ知識によって上達を感じられること
- ただし毎回まったく同じ最適ルートにはならないこと

## 方針: バイオーム単位ではある程度固定

完全ランダムではなく、バイオームや探索先の性格は固定する。

例:

- 廃坑エリア: 鉱石、金属素材、ピッケル関連素材が出やすい
- 森 / 菌糸洞窟エリア: 薬草、毒素材、回復薬素材が出やすい
- 崩れた砦エリア: 武器素材、防具素材、敵装備、高価な戦利品が出やすい
- 旧研究区画: レア素材、特殊クラフト素材が出るが危険度が高い

プレイヤーは「回復薬を作りたいから森へ行く」「剣を強化したいから砦へ行く」「鉱石が欲しいから廃坑へ行く」と判断できる。

## 固定するもの

- 探索先 / バイオームの名前
- その地域のテーマ
- 出やすい素材カテゴリ
- おおまかな危険度
- 代表的な敵
- 主要ランドマーク
- 地形の傾向

## ランダムにするもの

- 敵の配置や密度
- 採取ポイントや宝箱の具体的な位置
- レア素材の出現
- 一部の部屋や小区画の差し替え
- 通路の封鎖や崩落
- 脱出口の有効 / 無効
- 一時イベント
- 強敵や巡回敵の出現

## 目指す感触

プレイヤーには、目的地を選ぶ段階で計画を立てさせる。

例:

- 「今日は鉱石が欲しいから廃坑へ行く」
- 「回復薬の在庫が少ないから森で素材を集める」
- 「装備強化のために危険だが砦へ行く」

一方で、実際に現地で安全に回収して帰れるかは毎回変わる。

つまり、マップ知識は役に立つが、最終的な判断はその場の状況で変わる設計にする。

## 倉庫 / クラフト画面の方針

倉庫画面は、単なる保管場所ではなく、次の冒険の目的を決める作戦画面にしたい。

このゲームの中心が「必要なアイテムを集めるために冒険すること」なら、プレイヤーが倉庫を見ている時間に、自然と次のことを考えられる UI が望ましい。

- 何を持っているか
- 何が作れるか
- 何を作りたいか
- 何が足りないか
- 足りない素材はどこで手に入るか
- 次はどの探索先へ行くべきか

そのため、倉庫とクラフトは別画面に分けず、同じ画面で確認できる形がよさそう。

別画面に分けると、「倉庫を見る」「クラフトを見る」「足りない素材を覚える」「また倉庫を見る」という往復が発生し、考える楽しさより確認の面倒さが勝ちやすい。

### 画面構成案

左側に倉庫、右側にクラフト / レシピを置く。

倉庫側:

- 持っているアイテムを一度にたくさん見られるようにする
- マス目は現在より小さめにする
- 素材はアイコン、個数、カテゴリ色で素早く認識できるようにする
- 詳細説明は選択時やホバー時に出す

クラフト / レシピ側:

- 作れるもの、作れないものを一覧できる
- 必要素材ごとに所持数 / 必要数を表示する
- 足りている素材は明るく、足りない素材は目立つように表示する
- 足りない素材について、主な入手先を表示する

例:

- 鉄の剣 強化
- 鉱石: 2/5、あと3個必要、廃坑で入手しやすい
- 骨片: 1/1
- 200G: 120/200、あと80G必要

この表示があると、プレイヤーは自然に「次は廃坑へ行こう」と判断できる。

### 倉庫画面で起こしたい感情

倉庫画面では、次の3つを同時に起こしたい。

- 所有感: 素材が集まってきた満足感
- 不足感: あと何が足りないか分かる具体的な欲求
- 行き先の決定: 次にどこへ行けばよいか分かる納得感

この画面は、アイテムボックス、クラフト、素材辞典、次の目標選びが混ざった画面として考える。

ただし、最初から情報を出しすぎると重くなるため、初期表示は「作れる / 足りない / 入手先」を中心にし、詳細説明、派生レシピ、レア素材情報は選択時に出すくらいがよい。

## プロトタイプ用の具体案

このプロトタイプでは、まず「倉庫を見る」「作りたいものを選ぶ」「足りない素材を見る」「行き先を決める」という流れが成立することを優先する。

初期実装の物量は、バイオーム4種、素材24種、消耗品8種、装備 / 拠点強化レシピ20種、敵12種を目安にする。

数値バランスは後で調整する前提で、ここでは素材の役割と入手先が分かることを重視する。

- レア素材はただランダムに落ちるのではなく、危険な場所や強敵の近くに置く

### 入手先確認 UI に必要な情報

素材詳細では、次を表示したい。

- 主な入手先
- 入手方法
- 使用レシピ
- その素材を落とす敵
- レア度

レシピ詳細では、次を表示したい。

- 必要素材の所持数 / 必要数
- 足りない数
- 足りない素材の主な入手先
- その素材が取れる探索先への導線

例:

- ピッケル強化 I
- 鉄鉱石: 2/3、あと1個、廃坑の鉱脈
- 古い歯車: 0/1、あと1個、廃坑の機械残骸
- 木材: 4/2、足りている、全域の木箱

意図:

- プレイヤーに素材名を暗記させない
- 倉庫画面を見れば、自然に次の目的地が決まるようにする
- 「足りない素材」から「行き先」への導線を UI の中心にする
