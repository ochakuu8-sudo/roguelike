Create a 4x4 sprite atlas for a dark roguelike RPG. Exactly 16 equal invisible cells, one different sprite centered in each cell. This must be true small pixel art, not pixel-inspired illustration.

Most important: each asset must be designed as a native 32x32 game sprite.
Think of each cell's sprite as only 32 pixels wide by 32 pixels high at source resolution. The output may be shown large, but every visible source pixel should read as a deliberate square pixel. The design must have the simplicity of a real 32x32 RPG sprite: chunky silhouette, very few internal details, no high-resolution rendering, no detailed anime eyes, no tiny texture patterns, no complex material painting.

Do NOT create a detailed high-resolution chibi illustration. Do NOT create a painted icon and then pixelate it. Do NOT use a mosaic filter look. Instead, draw intentionally with large square pixel blocks, clear stair-step edges, simple 1-pixel-outline logic, and sparse clusters.

32x32 source-pixel constraints for every sprite:
- subject occupies about 22 to 28 source pixels tall, never more detailed than a 32x32 sprite
- dark outline is equivalent to a 1-pixel outline at source scale
- facial features are symbolic: one or two pixel clusters for eyes, one tiny mouth if needed
- each sprite uses about 4 to 6 colors total plus outline
- shadows are 2-to-5-pixel block clusters, not gradients
- highlights are one or two blocky clusters only
- no antialiasing, no smooth shading, no painterly texture, no semi-realistic lighting
- prioritize silhouette readability over detail
